# Visão do Projeto

## Problema

Iniciar um novo projeto frequentemente exige recriar a mesma infraestrutura:

* Configuração
* Banco de dados
* CRUD
* Autenticação
* Estrutura de pastas

Isso gera retrabalho.

## Solução

Criar um starter reutilizável em Go que sirva como base para:

* APIs REST
* MVPs
* Sistemas internos
* Microsserviços

## Resultado Esperado

Reduzir drasticamente o tempo necessário para iniciar novos projetos.

## Princípios

* Simplicidade
* Clareza
* Reutilização
* Aprendizado contínuo
